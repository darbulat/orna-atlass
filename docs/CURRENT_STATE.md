# ORNA Atlas: current implementation

Last verified: 2026-07-22. This document describes code that exists now. `ARCHITECTURE*.md` and `IMPLEMENTATION_PLAN_rus.md` may also describe target behavior; when they disagree, code, migrations and tests are authoritative.

## Capability matrix

| Area | State | Evidence / limitation |
|---|---|---|
| Public atlas, sessions and collections | Implemented | One public-location projection is used by location, atlas, search, session and collection flows. A migration-backed API test proves hidden exclusion and exact-coordinate non-disclosure across all of them. |
| Public homepage conversion journey | Implemented | The homepage opens directly on the interactive globe with the current dawn location selected, public preview playback starts from the single Listen action without an authentication gate, and the same Atlas component is used on `/` and `/atlas`. A minimal Map/Collections/About/Search/Sign in/Subscribe header remains above the globe. Compact popular-location previews and featured collections follow it; inline playback uses the persistent global player and Collections has a dedicated route. Globe, discovery, player, signup and explicit membership-intent actions emit only bounded event-name and placement pairs without personal data, URLs, coordinates or session identifiers. |
| Public legal information | Implemented | Public Privacy Policy and Terms of Use pages identify Kale Ltd., its BIN, registered address, CEO and registered IT activities; explain account, authentication, location, analytics, playback and acceptable-use practices; and are linked from the global footer and account flow. A browser canary verifies public routing, metadata, navigation and legal-entity disclosure. |
| PostgreSQL/PostGIS persistence | Implemented | Alembic revisions `0008`–`0015` add generated exact/public `geometry(Point,4326)`, GiST indexes, tombstones, cleanup jobs, pipeline state/leases, seed-link ownership and account-backed favorites/listening history. Bbox, anti-meridian and low-zoom clustering use PostGIS; dawn selection uses bounded KNN GiST scans across the target and adjacent wrapped meridians. A 100k-row integration test verifies bounded DB aggregation and spatial-index use. |
| Domain state constraints | Implemented | Shared enums validate API input and DB checks reject unknown states, invalid intervals and `approximate_public` locations without a public coordinate pair. Domain services raise transport-independent errors mapped at the FastAPI boundary. |
| Redis cache and RQ | Implemented | Mixed search has one stable SQL page; corrupt cache payloads are evicted. Invalidation is centralized after commit. Processing jobs persist stage status/attempt/error, request/RQ correlation IDs and heartbeats; RQ timeout/retry scales by declared duration, unknown-duration jobs use the safe maximum, and a recovery worker replaces stale queued/running jobs through RQ. |
| S3-compatible audio | Implemented | Masters/renditions are versioned and activation follows upload plus object verification. Ordered source segments can be packaged one WAV at a time as AAC/fMP4 HLS; immutable objects are verified and the manifest is published last. Cleanup jobs snapshot verified HLS inventories and delete only those exact keys; legacy single-object cleanup remains supported. |
| Playback | Implemented | Grants fail closed without an active ready stored rendition. Members-only sessions require an active entitlement and grants are audited. Private HLS objects are inventory-gated behind rendition-scoped expiring tokens signed with a dedicated production-required `HLS_TOKEN_SECRET`; key IDs and bounded previous-key verification support rotation independently of access-token signing. Legacy direct-file grants remain supported. The player uses native HLS or `hls.js`, refreshes grants and preserves position/state. |
| Coordinate privacy | Implemented | Exact and policy-derived public geometries are distinct; hidden rows have no public geometry. Public DTOs omit exact/public storage columns, internal metadata and service timestamps, while the admin DTO exposes editorial coordinates explicitly. |
| Authentication and membership | Implemented | Short-lived HS256 or RS256 access tokens, rotation-ready JWKS verification/publication, rotating server-side refresh tokens, secure cookies, self-service contracts and RBAC are present. E-mail/password, one-time 15-minute magic links delivered through configured SMTP, Google and Apple OIDC, and Facebook OAuth are supported. Magic-link and OAuth state is opaque, single-use and Redis-backed; magic-link tokens are stored only as SHA-256 digests and are bound to the initiating browser through a separate HttpOnly nonce cookie whose digest is kept with the token, so foreign browsers and mail scanners cannot consume the link. SMTP STARTTLS verifies the server certificate, and production rejects insecure or malformed callback transport. Social identities never auto-link to an existing e-mail account; OAuth user and identity rows are flushed in one service-owned transaction and no token issuance commit occurs until both exist. A free account does not imply membership access, and the UI truthfully states that enrollment and pricing are not open. Entitled users can discover and render members-only sessions. |
| Account library | Implemented | Favorites and listening history are synchronized to the authenticated account in PostgreSQL and exposed through bounded `/users/me` APIs and the `/library` UI. Public listening remains anonymous; failed or unauthenticated best-effort history writes never block playback. Members-only records resolve through the canonical entitlement policy and return not-found to non-entitled callers. Library DTOs omit coordinates, writes are idempotent, completion is forward-only, concurrent progress updates are ordered by the server-recorded request arrival time and services own transactions. |
| Admin authentication | Implemented with local compatibility | Production uses admin-role tokens and has an audited one-time first-admin command. The local header defaults off, is accepted only when explicitly enabled in local/development, and is rejected elsewhere. |
| BirdNET analysis | Implemented baseline | Provider output rejects non-finite/out-of-range intervals and confidence. The HLS worker analyzes each source independently, translates segment-local intervals onto the session timeline and commits successful siblings separately; a segment failure is recorded as partial without blocking HLS activation. A failed legacy attempt preserves `last_successful`. |
| Seed safety | Implemented | Seed requires explicit `--force` in local/test, takes a PostgreSQL advisory lock, claims only the exact owner marker, mutates only owned rows/links and never promotes a session without a ready stored rendition. Legacy manifest rows are adopted only with `--adopt-legacy-seed`; links are never inferred or claimed. |
| Frontend tests and session listening UI | Implemented | Eight Node-20 reducer/resource tests and one hundred two deterministic Playwright scenarios cover globe-first entry, capped-Atlas/dawn-location reconciliation at SSR and refresh time, current-dawn selection, locked-marker paywalls, inline/global player ownership, anonymous playback, malformed/unavailable Atlas fail-closed rendering, hidden-coordinate rejection, nullable Atlas contract fields, responsive touch targets, public navigation, conversion flows, typed errors, e-mail/social authentication, callback cleanup, bounded signup analytics, membership loading, outages and cross-session response races, shared single-flight access refresh across account/session/playback requests, grant refresh, switch/abort races, seek, cleanup and keyboard/ARIA behavior. Public session details aggregate BirdNET detections by species, expose bounded filtered episode lists with seek-to-audio actions, show allowlisted photo, altitude, temperature, wind, humidity and moon-phase context with truthful unavailable states, and keep pipeline assets collapsed behind technical details. |
| Frontend API contracts | Implemented | Sessions, atlas, dawn, auth, playback and collections consume `components["schemas"]` from generated OpenAPI types; CI regenerates and rejects drift. Public session, collection, media, annotation and BirdNET DTOs are explicit allowlists that omit arbitrary metadata, object keys, worker payloads and service timestamps. Typed errors distinguish auth, forbidden, not-found, conflict, 5xx, invalid response and network outage without invented fallback data. |
| Dependency integration tests | Implemented opt-in | Disposable PostgreSQL/PostGIS, Redis and MinIO tests cover migrations, constraints, 100k geospatial aggregation, public privacy/cache invalidation, mixed search pages, pipeline concurrency/partial failure/recovery, BirdNET savepoints, seed ownership and storage cleanup. |
| Observability | Implemented baseline | Structured request/worker logs carry request, processing-job and RQ-job IDs. API request logs record bounded route templates and the HTTPS gateway maps HLS paths to a redacted sentinel before access logging, so grant tokens are not written at either layer. API `/metrics` exposes bounded-label HTTP and privacy-bounded conversion metrics documented in [`CONVERSION_ANALYTICS.md`](CONVERSION_ANALYTICS.md); the worker serves fork-safe multiprocess queue, outcome and per-stage metrics on port `9101`. A hosted metrics backend and distributed tracing are deployment concerns, not implemented runtime claims. |
| Deployment | Implemented | API and ML worker use separate Dockerfiles. The API image installs neither development nor TensorFlow/BirdNET dependencies; CI builds and inspects both images. Compose gates API/workers on a one-shot successful migration service and gives long-running workers restart policies. |

## Runtime entry points

- API: `orna_atlas.app.main:app`.
- Worker: `python -m orna_atlas.app.workers.audio_pipeline worker`.
- Storage retention: `python -m orna_atlas.app.workers.storage_cleanup worker`.
- Stale pipeline recovery: `python -m orna_atlas.app.workers.pipeline_recovery worker`.
- Migrations: `alembic upgrade head`.
- Web: `web/package.json` scripts (`dev`, `build`, `test:e2e`).
- Local orchestration: `docker-compose.yml`.

## Known high-risk gaps

Cache invalidation is centralized and happens after commit, but remains best-effort rather than a durable outbox; a Redis outage can leave an entry stale until its short TTL. Repositories flush/query without committing and services own transaction boundaries.

The recorded 1–6 hour benchmark covers checksum/metadata and streaming waveform work on sparse 8 kHz PCM. It does not measure BirdNET, real S3 latency or production 44.1/48 kHz recordings; those measurements remain mandatory before defining production SLOs. Streaming renditions are still copied WAV files rather than HLS.

Playwright is deterministic and uses a mock API by default. `E2E_API_URL` can point it at a real stack, but CI does not yet run browser-to-database/media fixtures, an axe scan or real codec/network playback.

`media/service.py`, `AtlasExplorer.tsx` and global `styles.css` remain large. The session-detail schema still contains a substantial ORM hydration validator. Pure audio/player utilities and the shared required-null validator have been extracted; further component/pipeline/style decomposition is maintainability work, not a hidden production claim. `ARCHITECTURE.md` is canonical and `ARCHITECTURE_rus.md` is its translation.

## Updating this document

Move a capability to “Implemented” only when code plus an appropriate test proves it. Mention limitations explicitly. Planned product behavior belongs in architecture/plan documents, not in this current-state matrix.
